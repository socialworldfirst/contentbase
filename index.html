02The rules

1 · Neutral spine. Contentbase stores facts only: what exists, where it lives, what was said, how it performed. No scores, no verdicts, no opinions. Judgments (Brand Pull, boost picks, audience fits) are lenses — separate JSONs in their own projects that reference assets by asset_id.
2 · Stable ids. asset_id is assigned once and never changes (YouTube video id for anything on YouTube, IG post id otherwise). Lenses built last month still resolve next year.
3 · Scripts are the gold. Every YouTube asset carries its real spoken transcript (pulled from the channel's captions). Cross-posted IG reels inherit it. IG-only posts are caption-only until transcribed — their script_source says so honestly.
4 · Refresh adds, never deletes. A refresh appends new content, updates stats, and pairs new cross-posts. Existing pairings and ids are never reshuffled. Content removed from a platform keeps its record (status field for retiring later).
5 · Raw layers travel with it. Full transcripts (scripts/yt/) and downloaded IG thumbnails (thumbs/ig/ — IG links expire, ours don't) live next to the JSON, so the database is self-contained.

04How to call it

The database is one file: ~/Documents/Claude/Contentbase/contentbase.json. Read it directly, or use the helper:

# from the shell
python3 ~/Documents/Claude/Contentbase/api.py stats
python3 ~/Documents/Claude/Contentbase/api.py search "1688 supplier" --platform youtube
python3 ~/Documents/Claude/Contentbase/api.py get eVzw7d4qCEg
python3 ~/Documents/Claude/Contentbase/api.py newest 10
python3 ~/Documents/Claude/Contentbase/api.py export /tmp/assets.json   # base for a new lens

# from any Python / any Claude session
import sys; sys.path.insert(0, "/Users/steven/Documents/Claude/Contentbase")
from api import load, search, get, newest

In chat, just say "check the contentbase" / "search the contentbase for X" — any session can run the calls above. Registered in the Pipe Knowledge Index as contentbase, so other verticals discover it automatically.

05How to refresh

python3 ~/Documents/Claude/Contentbase/refresh.py          # full refresh
python3 ~/Documents/Claude/Contentbase/refresh.py --dry    # preview what would change

One command. It walks the live YouTube channel for new uploads (no dependency on any other pipeline), pulls their real transcripts, refreshes view counts, ingests new owned Instagram posts from the Meta export, pairs new cross-posts, stores thumbnails for new IG posts, and regenerates this page. Instagram freshness rides on the weekly /watched data dump (that's what updates the Meta export).

Not on a schedule yet, by choice. When wanted, a two-weekly scheduled task can run the same command — say the word.
